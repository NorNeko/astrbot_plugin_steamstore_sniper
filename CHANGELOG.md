# Changelog

## [v0.1.0] - 2026-04-24

首个可用版本，完成插件核心架构与全部基础功能。

### 新增功能

- **游戏详情查询** (`/steam`)：支持 AppID 纯数字输入、Steam 商店链接（含自定义 URL 及 `_` 占位符格式），并发拉取游戏详情与评测摘要，附带封面图。
- **多地区价格查询** (`/steam_price`)：指定任意 Steam 地区代码实时查询价格，折扣游戏同时展示原价与折扣力度。
- **评测语言区筛选**：通过 `language=` 参数区分简中 / 繁中 / 日语 / 英语 / 全部语言区好评率，精准反映各语言玩家评价。
- **会话级语言区切换** (`/steam_rlang`)：持久切换当前会话的评测统计语言，无需每次重复输入；单次查询可通过末尾内联语言代码临时覆盖。
- **URL 自动解析**：通过 `auto_parse_enabled` 开关 + ACL 白名单，在授权会话中发送商店链接即自动触发查询，无需指令前缀。
- **游戏截图查询** (`/steam_shots`)：批量拉取游戏截图，含成人内容防护（`block_adult_screenshots`）与数量限制（`max_screenshots`）。
- **访问控制（ACL）**：白名单 / 黑名单 / 关闭三种模式，所有指令与自动解析共用同一套 UMO 权限规则。

### 数据与输出

- 游戏分类标签使用 `genres`（开发商指定，如"独立、策略、抢先体验"），不使用 Steam 功能性标签（categories）。
- 输出格式采用 emoji 前缀行式排版，含标题（游戏名 + AppID）、简介、类型标签、开发商、发行商、价格、评测、发售日期、商店链接。
- 评测行根据评分动态选取氛围 emoji（🥳 / 😊 / 🤔 / 😞 / 😤）。

### 技术实现

- 三层架构：`steam_client`（HTTP 层）→ `store_service`（业务层）→ `formatter`（输出层）
- `asyncio.gather` 并发请求 appdetails + appreviews，评测失败不影响主流程
- `aiohttp` 会话复用，支持显式代理 URL 或 `trust_env` 环境变量代理
- 评测语言区三级优先级：内联参数 > 会话覆盖 > WebUI 全局默认
- 全部字段类型安全（dataclass + Optional），接口缺字段不抛异常

### 已知限制

- Steam 用户自定义标签（热门标签）需要 Steam 登录 Cookie，公开接口不可获取，已使用 `genres` 替代
- `appreviews` 的 `query_summary` 始终为全量全时段统计，无法单独获取"近期评测"好评率
